# Spring MVC Security - Dev Testing with Private/Incognito Windows

Hands On: `08-spring-mvc-security/code/01-spring-boot-mvc-security-default`

Spring securit is based on the user session

- Even when we close/open the tab or restart the spring server, the session is maintained
- This is only an issue during Dev and Testing
- Not an issue when deployed to Production / Real-Time

How to go about it?

- Start a new browser all-together, then quit the browser and start again every time
- Work with incognito/private windows
